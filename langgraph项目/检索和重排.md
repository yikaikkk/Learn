# 向量检索（Vector Search） 
1. 实现原理
- 技术栈 ：基于 PostgreSQL 的 pgvector 扩展
- 核心类 ： VectorSearcher （位于 kb_ingest/kb_service/services/search.py ）
- 实现方法 ：
  1. 将查询文本通过嵌入模型转换为向量
  2. 使用 PostgreSQL 的向量操作符计算相似度
  3. 按相似度排序并返回结果 2. 关键技术细节
- 距离度量 ：支持两种距离计算方式
  - 余弦相似度（cosine）：使用 <=> 操作符
  - 欧氏距离（l2）：使用 <-> 操作符
- 相似度计算 ：
  - 余弦相似度： 1.0 - (embedding <=> q.v)
  - 欧氏距离： 1.0 / (1.0 + (embedding <-> q.v))
- 性能优化 ：
  - 使用 SET ivfflat.probes = 10 优化 ivfflat 索引查询

# 二、重排（Rerank） 1. 实现原理
- 技术栈 ：支持两种重排服务
  - Cohere API：使用 Cohere 的重排模型
  - 自定义服务：调用自建的重排 API
- 核心类 ： RerankerClient （位于 kb_ingest/kb_service/services/reranker.py ）
- 实现方法 ：
  1. 收集初始检索结果作为候选
  2. 调用重排服务对候选进行评分
  3. 按重排分数排序结果 2. 关键技术细节
- 分数融合 ：支持向量分数和重排分数的融合
  - 使用 Z-score 标准化对分数进行归一化
  - 通过 fusion_alpha 参数控制权重分配
  - 计算公式： final_score = fusion_alpha * vec_z + (1 - fusion_alpha) * rerank_z
- 错误处理 ：实现了带重试的 API 调用，处理限流等问题
- 结果格式 ：统一返回 [{"id": str, "score": float}, ...] 格式

核心代码

```python
def rerank(self, query: str, candidates: Iterable[dict], top_n: Optional[int] = None) -> Optional[List[dict]]:
    # 准备文档列表
    documents = []
    for item in candidates:
        doc_id = str(item.get("id"))
        text = item.get("text") or item.get("content") or ""
        documents.append({"id": doc_id, "text": text})
    
    # 根据提供商调用不同的重排服务
    if self.mode == "cohere":
        return self._rerank_cohere(query, documents, top_n)
    else:
        return self._rerank_custom(query, documents)
```

# 完整检索流程
1. 完整流程
1. 向量检索 ：使用 pgvector 获取初始候选结果
2. 重排 ：对初始结果进行精细评估
3. 结果融合 ：
   - 优先返回向量检索和重排的交集结果
   - 按重排分数排序
   - 如果无交集，返回重排结果
   - 如果未启用重排，返回向量检索结果 2. 分数融合策略
- 标准化 ：使用 Z-score 对向量分数和重排分数进行归一化
- 加权融合 ：通过 fusion_alpha 参数控制两种分数的权重
- 排序 ：按融合后的最终分数排序结果 3. 配置参数
- 向量检索 ：
  - vector_top_k ：向量检索返回的候选数（默认 20）
  - threshold ：相似度阈值
- 重排 ：
  - rerank_enabled ：是否启用重排（默认 false）
  - rerank_top_k ：重排后返回的结果数（默认 10）
  - rerank_max_candidates ：重排的候选数量（默认 20）
  - rerank_score_fusion_alpha ：分数融合的权重（默认 0.3）
