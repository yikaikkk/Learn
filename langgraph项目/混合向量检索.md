# milvus和pgsql的混合向量检索

## 背景
在最近接触到的开源项目中，在rag部分使用了两种向量数据库，一种是pgsql结合pgvector，另一种是milvus

## 一、两个向量存储系统的实现 

1. Milvus 向量存储
- 实现文件 ： gustobot/infrastructure/knowledge/vector_store.py
- 核心类 ： VectorStore
- 功能 ：
  - 提供完整的向量数据库操作，包括创建集合、添加文档、搜索、删除等
  - 支持多种索引类型（如 IVF_FLAT）和距离度量方式（如 IP、L2、COSINE）
  - 支持元数据过滤
  - 集成到 KnowledgeService 中，用于菜谱知识库的检索 
  
2. PostgreSQL (pgvector) 向量存储
- 实现文件 ： kb_ingest/kb_service/services/search.py
- 核心类 ： VectorSearcher
- 功能 ：
  - 使用 pgvector 扩展进行向量相似度搜索
  - 支持余弦相似度和欧氏距离计算
  - 提供混合检索功能（向量检索 + 重排序）
  - 支持公司名称和来源表过滤

## 二、混合检索的实现方式
项目中的混合检索主要体现在以下几个方面：
 1. 系统级混合
- Milvus ：用于菜谱知识库的向量检索（ KnowledgeService ）
- PostgreSQL (pgvector) ：用于通用知识库的向量检索（ VectorSearcher ）
- 使用场景 ：根据不同的业务需求选择合适的向量存储系统 2. 单系统内的混合检索
- PostgreSQL 内的混合 ：
  
  - 向量检索：使用 pgvector 进行初始相似度搜索
  - 重排序：使用重排模型对初始结果进行优化
  - 分数融合：支持向量分数和重排分数的加权融合
- Milvus 内的混合 ：
  
  - 向量检索：使用 Milvus 进行初始相似度搜索
  - 重排序：使用重排模型对初始结果进行优化
  - 过滤：支持基于相似度阈值和重排分数阈值的过滤

## 三、使用流程 
1. PostgreSQL (pgvector) 检索流程
    1. 向量检索 ：使用 pgvector 计算查询向量与文档向量的相似度
    2. 重排序 ：对初始结果使用重排模型进行精细评估
    3. 分数融合 ：如果启用，融合向量分数和重排分数
    4. 结果返回 ：返回排序后的结果 

2. Milvus 检索流程
    1. 向量检索 ：使用 Milvus 计算查询向量与文档向量的相似度
    2. 过滤 ：根据相似度阈值过滤结果
    3. 重排序 ：对过滤后的结果使用重排模型进行优化
    4. 结果返回 ：返回排序后的结果

## 四、适用场景
### 1. PostgreSQL (pgvector) 适用场景 通用文档检索
- 数据来源 ：从 Excel 文件或 MySQL 数据库导入的通用数据
- 数据类型 ：结构化或半结构化数据，如公司信息、报告、表格数据等
- 检索方式 ：
  - 通过 REST API 端点（ /search 和 /search/hybrid ）提供服务
  - 支持公司名称过滤（ company_filter ）和来源表过滤（ source_tables ）
  - 适合需要结合关系型数据的场景 技术特点
- 关系型数据库集成 ：与 PostgreSQL 关系型数据无缝集成
- 灵活查询 ：支持 SQL 查询和 JSONB 字段查询
- 中小规模数据 ：适合数据量适中的场景
- 部署简单 ：作为 PostgreSQL 扩展，部署和维护相对简单
### 2. Milvus 适用场景 菜谱知识库检索
- 数据来源 ：内部菜谱数据，通过 KnowledgeService 导入
- 数据类型 ：菜谱相关数据，包括菜名、分类、食材、步骤等
- 检索方式 ：
  - 通过 KnowledgeService.search() 方法提供内部服务
  - 支持基于菜谱属性的过滤（如分类、难度）
  - 适合专门的领域知识库场景 技术特点
- 高性能 ：作为专门的向量数据库，检索性能更优
- 可扩展性 ：支持水平扩展，适合大规模向量数据
- 丰富的索引 ：支持多种索引类型（如 IVF_FLAT、HNSW）
- 专业向量管理 ：提供完整的向量数据库功能，如向量索引、相似度搜索等
