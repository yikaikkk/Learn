# 项目复盘-Clickhouse-part2

## 背景
在part1中，数据从kafka中处理以后会存入到clickhouse集群。在数据量巨大，并且涉及到不同业务的时候，就涉及到分库分表

## 分库分表
分库-分库是指在表数量不变的情况下对库进行切分。

分表-分表是指在库数量不变的情况下对表进行切分。

### 分库分表策略
1、水平切分

水平切分包含水平分库和水平分表。

1.1 水平分表

水平分表指的表结构不变，将单表数据切分成多表。切分后的结果：每个表的结构一样；
每个表的数据不一样；
所有表的数据并集为全量数据；

1.2 水平分库

水平分库是指，将表水平切分后分到不同的数据库，使得每个库具有相同的表，表中的数据不相同。

2、垂直切分

垂直切分包含垂直分库和垂直分表。

2.1 垂直分表

垂直分表指将存在一张表中的字段切分到多张表。切分后的结果：

每个表的结构不一样；每个表的数据不一样；所有表的字段并集是原表的字段；

2.2 垂直分库

垂直分库指的是，将单个库中的表分到多个库，每个库包含的表不一样。
